PROJECT SUMMARY AND CONCLUSION
Date: 2026-03-22

1) OBJECTIVE
Build a date-aware machine learning pipeline to predict plant parameters per plant record using sensor readings.
Targets:
- height
- length
- weight
- leaves
- branches

Important requirement:
- Split by date (chronological), not by random lines.
- Keep plant identity in features (plant_no, plant_system).

2) DATASET USED
Primary dataset:
- COMBINED_SENSOR_PLANT_DATA_WITH_LENGTH.csv

Dataset profile:
- Rows: 384
- Columns: 14
- Date range: 2026-02-06 to 2026-03-17
- Systems: AERO and DWC (balanced)
- plant_no: 1 to 6
- Missing values: none
- Duplicate rows: none

3) DATE-BASED SPLIT CREATED
Chronological split based on unique dates (70/30):
- training_data.csv
  - Date range: 2026-02-06 to 2026-03-06
  - Rows: 276
- test_data.csv
  - Date range: 2026-03-07 to 2026-03-17
  - Rows: 108

Detailed split method (important):
- Source file: COMBINED_SENSOR_PLANT_DATA_WITH_LENGTH.csv
- Date parsing format: yyyy-mm-dd
- Split key: unique date values (not row order)
- Total unique dates: 32
- Train date count: 23 dates (70% of 32 rounded up with ceiling)
- Test date count: 9 dates
- Date cutoff point:
  - Last train date: 2026-03-06
  - First test date: 2026-03-07
- No date overlap between training_data.csv and test_data.csv
- Rows per date are consistent (12 records/date), so row split is:
  - Train: 23 x 12 = 276 rows
  - Test: 9 x 12 = 108 rows

Leakage-control checks performed:
- Train and test are chronologically separated.
- Plant identifiers were preserved in both sets (plant_system and plant_no).
- No random shuffling was used for the split.

Inference-style test input (sensor-only):
- test_this.csv
  - Contains date/day/plant_system/plant_no + sensor columns only
  - Target columns removed: height, length, weight, leaves, branches
  - Row count: 108 (same rows as test_data.csv)

Data files generated in order:
- COMBINED_SENSOR_PLANT_DATA_WITH_LENGTH.csv (input)
- training_data.csv (date-based train split)
- test_data.csv (date-based test split)
- test_this.csv (sensor-only inference file from test_data.csv)

4) MODELING APPROACH
Baseline training script:
- train_best_models_and_predict.py

Feature strategy:
- Sensor features: ave_ph, ave_do, ave_tds, ave_temp, ave_humidity
- Plant identity/context: plant_no, plant_system
- Date-derived features: day_of_year, day_of_week_num, week_of_year

Candidate models tested per target:
- LinearRegression
- Ridge (multiple alpha values)
- RandomForest
- ExtraTrees
- GradientBoosting

Validation strategy:
- TimeSeriesSplit (date-aware CV) to reduce temporal leakage.

5) BASELINE BEST MODELS
From best_models_summary.csv:
- height: ExtraTrees_300
- length: ExtraTrees_300
- weight: ExtraTrees_300
- leaves: GradientBoosting_300
- branches: ExtraTrees_300

Best-fit conclusion from model search:
- ExtraTrees is the best-fit family overall in this project.
- It won 4 of 5 targets in baseline training.
- GradientBoosting was best-fit only for leaves in baseline.

6) IMPROVEMENT RUN

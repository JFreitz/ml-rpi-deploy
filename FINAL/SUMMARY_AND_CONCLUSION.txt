FINAL PROJECT SUMMARY AND CONCLUSION
Date: 2026-03-22

A) PROJECT GOAL
Predict plant parameters per record using sensor readings while preserving temporal order and plant identity.
Predicted targets:
- height
- length
- weight
- leaves
- branches

Key constraints followed:
- Split by date (chronological), not by random row.
- Keep plant identity/context (plant_no, plant_system).
- Evaluate on future dates only to reduce leakage.


B) INPUT DATASET
Source file:
- COMBINED_SENSOR_PLANT_DATA_WITH_LENGTH.csv

Dataset profile:
- Rows: 384
- Columns: 14
- Date range: 2026-02-06 to 2026-03-17
- Unique dates: 32
- Systems: AERO (192 rows), DWC (192 rows)
- plant_no: 1..6
- Missing values: none
- Duplicate rows: none
- Duplicate date+plant_system+plant_no keys: none

Columns:
- date, day, plant_system
- ave_ph, ave_do, ave_tds, ave_temp, ave_humidity
- plant_no
- height, length, weight, leaves, branches


C) SPLIT METHOD (FULL DETAIL)
Split logic used:
- Parse date as yyyy-mm-dd
- Sort by unique date values
- 70/30 chronological split by unique dates
- Use ceiling for train date count

Computed split:
- Total unique dates: 32
- Train unique dates: 23 (ceil(32 * 0.70))
- Test unique dates: 9

Date boundaries:
- training_data.csv: 2026-02-06 to 2026-03-06
- test_data.csv: 2026-03-07 to 2026-03-17

Rows:
- training_data.csv: 276
- test_data.csv: 108

Additional inference file:
- test_this.csv (sensor-only from test_data.csv)
- Kept: date, day, plant_system, plant_no, ave_ph, ave_do, ave_tds, ave_temp, ave_humidity
- Removed targets: height, length, weight, leaves, branches

Leakage controls:
- No date overlap between train and test.
- No random shuffle during split.
- Plant identifiers preserved in both sets.

Base features:

G) BEST-FIT MODEL CONCLUSION
Overall best-fit family across runs:
- ExtraTrees performed best for most targets and configurations.

Important exception:
- GradientBoosting often won for leaves (and some DWC/system-specific targets).

Operational conclusion:
- Best practical setup is a mixed strategy, not a single algorithm for all targets.


H) RECOMMENDED DEPLOYMENT OUTPUT
Use this prediction file in production:

  (built as mixed: F2 for all targets, F1 for leaves)

Why:
- Keeps stronger system-split-all performance for height, length, weight, branches.
